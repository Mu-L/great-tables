Column Selection

The columns= argument for methods like tab_spanner() and cols_move() can accept a range of arguments. In the previous examples, we just passed a list of strings with the exact column names. However, we can specify columns using any of the following:

  • a single string column name.
  • an integer for the column’s position.
  • a list of strings or integers.
  • a Polars selector.
  • a function that takes a string and returns True or False.
from great_tables import GT
from great_tables.data import exibble

gt_ex = GT(exibble)

String and Integer Selectors

We can use a list of strings or integers to select columns by name or position, respectively.

gt_ex.cols_move_to_start(columns=["date", 1, -1])
date char group num fctr time datetime currency row
2015-01-15 apricot grp_a 0.1111 one 13:35 2018-01-01 02:22 49.95 row_1
2015-02-15 banana grp_a 2.222 two 14:40 2018-02-02 14:33 17.95 row_2
2015-03-15 coconut grp_a 33.33 three 15:45 2018-03-03 03:44 1.39 row_3
2015-04-15 durian grp_a 444.4 four 16:50 2018-04-04 15:55 65100.0 row_4
2015-05-15 grp_b 5550.0 five 17:55 2018-05-05 04:00 1325.81 row_5
2015-06-15 fig grp_b six 2018-06-06 16:11 13.255 row_6
grapefruit grp_b 777000.0 seven 19:10 2018-07-07 05:22 row_7
2015-08-15 honeydew grp_b 8880000.0 eight 20:20 0.44 row_8

Note the code above moved the following columns:

  • The string "date" matched the column of the same name.
  • The integer 1 matched the second column (this is similar to list indexing).
  • The integer -1 matched the last column.

Moreover, the order of the list defines the order of selected columns. In this case, "data" was the first entry, so it’s the very first column in the new table.

Using Function Selectors

Polars Selectors

When using a Polars DataFrame, you can select columns using Polars selectors. The example below uses Polars selectors to move all columns that start with "c" or "f" to the start of the table.

import polars as pl
import polars.selectors as cs

pl_df = pl.from_pandas(exibble)

GT(pl_df).cols_move_to_start(columns=cs.starts_with("c") | cs.starts_with("f"))
char currency fctr num date time datetime row group
apricot 49.95 one 0.1111 2015-01-15 13:35 2018-01-01 02:22 row_1 grp_a
banana 17.95 two 2.222 2015-02-15 14:40 2018-02-02 14:33 row_2 grp_a
coconut 1.39 three 33.33 2015-03-15 15:45 2018-03-03 03:44 row_3 grp_a
durian 65100.0 four 444.4 2015-04-15 16:50 2018-04-04 15:55 row_4 grp_a
None 1325.81 five 5550.0 2015-05-15 17:55 2018-05-05 04:00 row_5 grp_b
fig 13.255 six None 2015-06-15 None 2018-06-06 16:11 row_6 grp_b
grapefruit None seven 777000.0 None 19:10 2018-07-07 05:22 row_7 grp_b
honeydew 0.44 eight 8880000.0 2015-08-15 20:20 None row_8 grp_b

In general, selection should match the behaviors of the Polars DataFrame.select() method.

pl_df.select(cs.starts_with("c") | cs.starts_with("f")).columns
['char', 'currency', 'fctr']

See the Selectors page in the polars docs for more information on this.
